================================
IndustrialComplexParentStructure
================================

Overview
========

.. rubric:: Abstract class

A :doc:`PlantStructureItem </reference/PlantStructure/PlantStructureItem>` that is a suitable :ref:`ParentStructure <dexpi-PlantStructure-IndustrialComplex-ParentStructure>` of an :doc:`IndustrialComplex </reference/PlantStructure/IndustrialComplex>`.

.. image:: IndustrialComplexParentStructure.*


.. rubric:: Subtypes

* :doc:`Enterprise </reference/PlantStructure/Enterprise>`

* :doc:`Site </reference/PlantStructure/Site>`

.. admonition:: proteus

   Implementation is subclass-specific.

.. admonition:: example

   As :doc:`IndustrialComplexParentStructure </reference/PlantStructure/IndustrialComplexParentStructure>` is abstract, we consider :doc:`Enterprise </reference/PlantStructure/Enterprise>` as an arbitrary concrete subclass.

   .. image:: /diagrams/diagram733.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <PlantStructureItem
            ID="enterprise1"
            ComponentClass="Isa95Enterprise"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS10418236543" DOTS123="DOTS123">
          ...
        </PlantStructureItem>
