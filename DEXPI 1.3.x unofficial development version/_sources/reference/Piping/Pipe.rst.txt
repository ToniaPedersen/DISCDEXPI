====
Pipe
====

Overview
========

.. rubric:: Class

An elementary piece of piping, i.e., not interrupted by any item.

.. image:: Pipe.*


.. rubric:: Supertypes

* :doc:`ConceptualObject </reference/DexpiModel/ConceptualObject>`

* :doc:`CustomAttributeOwner </reference/Customization/CustomAttributeOwner>`

* :doc:`PipingConnection </reference/Piping/PipingConnection>`

.. admonition:: proteus

   A :doc:`Pipe </reference/Piping/Pipe>` is implemented as a ``<CenterLine>`` element within a ``<PipingNetworkSegment>`` element.

   The :ref:`SourceItem <dexpi-Piping-PipingConnection-SourceItem>`, :ref:`SourceNode <dexpi-Piping-PipingConnection-SourceNode>`, :ref:`TargetItem <dexpi-Piping-PipingConnection-TargetItem>`, and :ref:`TargetNode <dexpi-Piping-PipingConnection-TargetNode>` attributes inherited from :doc:`PipingConnection </reference/Piping/PipingConnection>` are not directly implemented in Proteus Schema. They are rather given implicitly by the order of ``<CenterLine>`` and other elements in the ``<PipingNetworkSegment>``. For details, see the Proteus Schema specification.

.. admonition:: example

   .. image:: /diagrams/diagram671.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <PipingNetworkSegment
            ID="pipingNetworkSegment1"
            ComponentClass="PipingNetworkSegment"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS267704">
          ...
          <!--
            Only  a <CenterLine> whose parent is a <PipingNetworkSegment>
            implements a DEXPI Pipe.
          -->
          <CenterLine DOTS123="DOTS123">
            ...
          </CenterLine>
          ...
        </PipingNetworkSegment>
