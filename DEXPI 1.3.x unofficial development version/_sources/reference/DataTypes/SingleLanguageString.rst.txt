====================
SingleLanguageString
====================

Overview
========

.. rubric:: Data type

A :doc:`SingleLanguageString </reference/DataTypes/SingleLanguageString>` contains a :doc:`NullableString </reference/DataTypes/NullableString>` as its :ref:`Value <dexpi-DataTypes-SingleLanguageString-Value>` and a :ref:`Language <dexpi-DataTypes-SingleLanguageString-Language>` tag. :doc:`SingleLanguageString </reference/DataTypes/SingleLanguageString>` is only used within :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>`. See the latter data type for more details.

.. _dexpi-DataTypes-SingleLanguageString-Language:

Language
========

.. rubric:: Attribute (data)


The language of the :ref:`Value <dexpi-DataTypes-SingleLanguageString-Value>` of the :doc:`SingleLanguageString </reference/DataTypes/SingleLanguageString>`. If not :doc:`NULL_STRING </reference/DataTypes/NULL_STRING>`, the language must be given as a language tag standardized by IETF BCP 47 (*Best Current Practice 47* by the *Internet Engineering Task Force*)DOTANDSPACEHACKIETF BCP 47 is based on ISO 639.

In DEXPI, only those language tags are allowed

* that are listed in the normative registry file at `https://www.iana.org/assignments/language-subtag-registry/language-subtag-registry <https://www.iana.org/assignments/language-subtag-registry/language-subtag-registry>`__ from September 29, 2020,

* that consist of exactly two letters (in particular extended tags such as *en-US* or *de-CH-1996* are not allowed),

* that refer to an actual language (the *type* defined by IETF BCP 47 must be *language*),

* and that are not deprecated.
