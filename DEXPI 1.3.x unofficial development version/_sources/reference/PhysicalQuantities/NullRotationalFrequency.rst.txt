=======================
NullRotationalFrequency
=======================

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of application type :doc:`NullableRotationalFrequency </reference/PhysicalQuantities/NullableRotationalFrequency>`. The only instance of this singleton type is :doc:`NULL_ROTATIONAL_FREQUENCY </reference/PhysicalQuantities/NULL_ROTATIONAL_FREQUENCY>`.

.. image:: NullRotationalFrequency.*


.. rubric:: Supertypes

* :doc:`NullableRotationalFrequency </reference/PhysicalQuantities/NullableRotationalFrequency>`

.. admonition:: proteus

   All data attributes with type :doc:`NullableRotationalFrequency </reference/PhysicalQuantities/NullableRotationalFrequency>` (the base type of :doc:`NullRotationalFrequency </reference/PhysicalQuantities/NullRotationalFrequency>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram600.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_ROTATIONAL_FREQUENCY </reference/PhysicalQuantities/NULL_ROTATIONAL_FREQUENCY>` (e.g., see the implementation example for the attribute :ref:`DesignRotationalSpeed <dexpi-Equipment-Agglomerator-DesignRotationalSpeed>` of the DEXPI class :doc:`Agglomerator </reference/Equipment/Agglomerator>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
