.. admonition:: proteus

   All data attributes with type :doc:`NumberOfPortsClassification </reference/Enumerations/NumberOfPortsClassification>` are implemented as :ref:`DEXPI generic attributes for enumeration values <proteus-concept-dexpi-generic-attribute-enumeration>`. In a ``<GenericAttribute>`` element, the :doc:`NumberOfPortsClassification </reference/Enumerations/NumberOfPortsClassification>` literal is given by means of its :ref:`RDL reference <proteus-concept-rdl-reference>` in the table above. The ``Value`` attribute of the element is the name of the RDL reference in camel-case; it equals the name of the literal. The ``ValueURI`` attribute of the element is the URI of the RDL reference.

.. admonition:: example

   .. image:: /diagrams/diagram63.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="anyURI"
          Value="TwoPortValve"
          ValueURI="http://data.posccaesar.org/rdl/RDS11506315" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the instance :ref:`TwoPortValve <dexpi-Enumerations-NumberOfPortsClassification-TwoPortValve>`. For a complete example, see the attribute :ref:`NumberOfPorts <dexpi-Piping-OperatedValve-NumberOfPorts>` of the DEXPI class :doc:`OperatedValve </reference/Piping/OperatedValve>`.
