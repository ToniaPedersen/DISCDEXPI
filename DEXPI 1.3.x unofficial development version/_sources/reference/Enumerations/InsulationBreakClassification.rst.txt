.. admonition:: proteus

   All data attributes with type :doc:`InsulationBreakClassification </reference/Enumerations/InsulationBreakClassification>` are implemented as :ref:`DEXPI generic attributes for enumeration values <proteus-concept-dexpi-generic-attribute-enumeration>`. In a ``<GenericAttribute>`` element, the :doc:`InsulationBreakClassification </reference/Enumerations/InsulationBreakClassification>` literal is given by means of its :ref:`RDL reference <proteus-concept-rdl-reference>` in the table above. The ``Value`` attribute of the element is the name of the RDL reference in camel-case; it equals the name of the literal. The ``ValueURI`` attribute of the element is the URI of the RDL reference.

.. admonition:: example

   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="anyURI"
          Value="InsulationBreak"
          ValueURI="http://sandbox.dexpi.org/rdl/InsulationBreak" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the instance :ref:`InsulationBreak <dexpi-Enumerations-InsulationBreakClassification-InsulationBreak>`. For a complete example, see the attribute :ref:`InsulationBreak <dexpi-Piping-PropertyBreak-InsulationBreak>` of the DEXPI class :doc:`PropertyBreak </reference/Piping/PropertyBreak>`.
