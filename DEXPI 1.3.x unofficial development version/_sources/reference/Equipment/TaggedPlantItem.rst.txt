.. _dexpi-Equipment-TaggedPlantItem-TagName:

TagName
=======

.. rubric:: Attribute (data)


The tag number of the :doc:`TaggedPlantItem </reference/Equipment/TaggedPlantItem>`. See also :ref:`TagNamePrefix <dexpi-Equipment-TaggedPlantItem-TagNamePrefix>`, :ref:`TagNameSequenceNumber <dexpi-Equipment-TaggedPlantItem-TagNameSequenceNumber>`, and :ref:`TagNameSuffix <dexpi-Equipment-TaggedPlantItem-TagNameSuffix>`.

Multiplicity
    0..1

Type
    :doc:`NullableString </reference/DataTypes/NullableString>`

.. admonition:: proteus

   The attribute is implemented as a :ref:`DEXPI generic attribute for string values <proteus-concept-dexpi-generic-attribute-string>`.

   RDL reference
       `TAG NAME ASSIGNMENT CLASS <http://sandbox.dexpi.org/rdl/TagNameAssignmentClass>`__

   Name
       TagNameAssignmentClass

   AttributeURI
       `http://sandbox.dexpi.org/rdl/TagNameAssignmentClass <http://sandbox.dexpi.org/rdl/TagNameAssignmentClass>`__

.. admonition:: example

   As the owning class :doc:`TaggedPlantItem </reference/Equipment/TaggedPlantItem>` is abstract, we consider :doc:`Vessel </reference/Equipment/Vessel>` as an arbitrary concrete subclass.

   "P4714-A" (:doc:`String </reference/DataTypes/String>`\ )

   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="vessel1"
            ComponentClass="Vessel"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS414674" DOTS123="DOTS123">
          ...
          <GenericAttributes Set="DexpiAttributes" DOTS123="DOTS123">
            <GenericAttribute
              Name="TagNameAssignmentClass"
              AttributeURI="http://sandbox.dexpi.org/rdl/TagNameAssignmentClass"
              Format="string"
              Value="P4714-A" />
            ...
          </GenericAttributes>
          ...
        </Equipment>
      


.. _dexpi-Equipment-TaggedPlantItem-TagNamePrefix:

TagNamePrefix
=============

.. rubric:: Attribute (data)


The prefix part of the tag number of the :doc:`TaggedPlantItem </reference/Equipment/TaggedPlantItem>`. For example, the prefix of the tag number "P4714-A" is "P". The prefix often indicates the type of the equipment item, e.g., "P" can indicate a pump. See also :ref:`TagName <dexpi-Equipment-TaggedPlantItem-TagName>`.

Multiplicity
    0..1

Type
    :doc:`NullableString </reference/DataTypes/NullableString>`

.. admonition:: proteus

   The attribute is implemented as a :ref:`DEXPI generic attribute for string values <proteus-concept-dexpi-generic-attribute-string>`.

   RDL reference
       `TAG NAME PREFIX ASSIGNMENT CLASS <http://sandbox.dexpi.org/rdl/TagNamePrefixAssignmentClass>`__

   Name
       TagNamePrefixAssignmentClass

   AttributeURI
       `http://sandbox.dexpi.org/rdl/TagNamePrefixAssignmentClass <http://sandbox.dexpi.org/rdl/TagNamePrefixAssignmentClass>`__

.. admonition:: example

   As the owning class :doc:`TaggedPlantItem </reference/Equipment/TaggedPlantItem>` is abstract, we consider :doc:`Vessel </reference/Equipment/Vessel>` as an arbitrary concrete subclass.

   "P" (:doc:`String </reference/DataTypes/String>`\ )

   .. admonition:: proteus-example
   
      
      .. code-block:: xml

            ID="vessel1"
            ComponentClass="Vessel"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS414674" DOTS123="DOTS123">
