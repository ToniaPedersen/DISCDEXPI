Overview
========

The :doc:`PlantStructure </reference/PlantStructure/index>` package provides classes to assign :doc:`TechnicalItems </reference/PlantStructure/TechnicalItem>` (e.g., :doc:`TaggedPlantItems </reference/Equipment/TaggedPlantItem>` or :doc:`PipingNetworkSystems </reference/Piping/PipingNetworkSystem>`) to a hierarchy of structures, i.e., technical or organizational groups.

Main Hierarchy
--------------

The main hierarchy consists of 6 classes:

- :doc:`Enterprise </reference/PlantStructure/Enterprise>`

- :doc:`Site </reference/PlantStructure/Site>`

- :doc:`IndustrialComplex </reference/PlantStructure/IndustrialComplex>`

- :doc:`ProcessPlant </reference/PlantStructure/ProcessPlant>`

- :doc:`PlantSection </reference/PlantStructure/PlantSection>`

- :doc:`TechnicalItem </reference/PlantStructure/TechnicalItem>` (abstract superclass of :doc:`TaggedPlantItem </reference/Equipment/TaggedPlantItem>`, :doc:`PipingNetworkSystem </reference/Piping/PipingNetworkSystem>`, etc.)

For each object of one of these classes (except :doc:`Enterprise </reference/PlantStructure/Enterprise>`), a parent structure can be given. A parent structure must be an instance of a class that is on a higher level in the hierarchy. For example, only an :doc:`Enterprise </reference/PlantStructure/Enterprise>` is a suitable parent structure of a :doc:`Site </reference/PlantStructure/Site>`. The parent structure of an 
:doc:`IndustrialComplex </reference/PlantStructure/IndustrialComplex>` can be an :doc:`Enterprise </reference/PlantStructure/Enterprise>` or a :doc:`Site </reference/PlantStructure/Site>`.

In order to capture these restrictions in the information model, some abstract auxiliary classes are used. For example, :doc:`IndustrialComplexParentStructure </reference/PlantStructure/IndustrialComplexParentStructure>` is an abstract base class of both :doc:`Enterprise </reference/PlantStructure/Enterprise>` and :doc:`Site </reference/PlantStructure/Site>`. It is used as the type of the :ref:`ParentStructure <dexpi-PlantStructure-IndustrialComplex-ParentStructure>` attribute of :doc:`IndustrialComplex </reference/PlantStructure/IndustrialComplex>`:

Side Hierarchies
----------------

In addition to the main hierarchy, there are 3 side hierarchies:

- Instances of :doc:`IndustrialComplex </reference/PlantStructure/IndustrialComplex>`, :doc:`ProcessPlant </reference/PlantStructure/ProcessPlant>`, :doc:`PlantSection </reference/PlantStructure/PlantSection>`, and :doc:`TechnicalItem </reference/PlantStructure/TechnicalItem>` can be assigned to an :doc:`PlantArea </reference/PlantStructure/PlantArea>`, see :doc:`PlantAreaLocatedStructure </reference/PlantStructure/PlantAreaLocatedStructure>`.

- :doc:`TechnicalItems </reference/PlantStructure/TechnicalItem>` can be assigned to a :doc:`PlantSystem </reference/PlantStructure/PlantSystem>`, see :doc:`PlantSystemLocatedStructure </reference/PlantStructure/PlantSystemLocatedStructure>`.

- :doc:`TechnicalItems </reference/PlantStructure/TechnicalItem>` can be assigned to a :doc:`PlantTrain </reference/PlantStructure/PlantTrain>`, see :doc:`PlantTrainLocatedStructure </reference/PlantStructure/PlantTrainLocatedStructure>`.
