===============
NullTemperature
===============

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullableTemperature </reference/PhysicalQuantities/NullableTemperature>`. The only instance of this singleton type is :doc:`NULL_TEMPERATURE </reference/PhysicalQuantities/NULL_TEMPERATURE>`.

.. image:: NullTemperature.*


.. rubric:: Supertypes

* :doc:`NullableTemperature </reference/PhysicalQuantities/NullableTemperature>`

.. admonition:: proteus

   All data attributes with type :doc:`NullableTemperature </reference/PhysicalQuantities/NullableTemperature>` (the base type of :doc:`NullTemperature </reference/PhysicalQuantities/NullTemperature>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram601.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_TEMPERATURE </reference/PhysicalQuantities/NULL_TEMPERATURE>` (e.g., see the implementation example for the attribute :ref:`LowerLimitDesignTemperature <dexpi-Equipment-Chamber-LowerLimitDesignTemperature>` of the DEXPI class :doc:`Chamber </reference/Equipment/Chamber>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
