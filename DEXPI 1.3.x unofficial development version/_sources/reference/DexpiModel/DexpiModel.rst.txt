.. admonition:: proteus

   The class is implemented using the Proteus root element ``<PlantModel>``, i.e., a :doc:`DexpiModel </reference/DexpiModel/DexpiModel>` corresponds to an entire Proteus XML document.

   Note that Proteus Schema requires that the ``<PlantModel>`` element contains a ``<PlantInformation>`` element. The latter has several required attributes:

   * ``Application``: fixed value ``Dexpi`` (optional in Proteus Schema, but required by DEXPI);

   * ``ApplicationVersion``: fixed value ``1.3`` (optional in Proteus Schema, but required by DEXPI);

   * ``Date``: see :ref:`ExportDateTime <dexpi-DexpiModel-DexpiModel-ExportDateTime>`;

   * ``Discipline``: fixed value ``PID``;

   * ``Is3D``: fixed value ``no``;

   * ``OriginatingSystem``: see :ref:`OriginatingSystemName <dexpi-DexpiModel-DexpiModel-OriginatingSystemName>`;

   * ``SchemaVersion``: fixed value ``4.1``;

   * ``Time``: see :ref:`ExportDateTime <dexpi-DexpiModel-DexpiModel-ExportDateTime>`;

   The ``<PlantInformation>`` element must contain a ``<UnitsOfMeasure>`` element. However, none of the attributes of ``<UnitsOfMeasure>`` is relevant for DEXPI.

.. admonition:: example

   .. image:: /diagrams/diagram43.*


   .. admonition:: proteus-example
   
      The XML fragment below only shows the fixed XML attributes of the ``<PlantInformation>`` element. For the other required XML attributes ``Date``, ``OriginatingSystem``, ``Time``, and ``Units``, see the Proteus Schema implementations of the DEXPI attributes given above.

      
      .. code-block:: xml
      
        <PlantModel>
          <PlantInformation
              Discipline="PID"
              Is3D="no"
              SchemaVersion="4.1"
              DOTS123="DOTS123">
            <UnitsOfMeasure/>
          </PlantInformation>
        </PlantModel>

.. _dexpi-DexpiModel-DexpiModel-ExportDateTime:

ExportDateTime
==============

.. rubric:: Attribute (data)


The date time at which the :doc:`DexpiModel </reference/DexpiModel/DexpiModel>` was exported by the originating system (see :ref:`OriginatingSystemName <dexpi-DexpiModel-DexpiModel-OriginatingSystemName>`\ ).

Multiplicity
    1

Type
    :doc:`NullableDateTime </reference/DataTypes/NullableDateTime>`

.. admonition:: proteus

   The attribute is implemented using the XML attributes ``Date`` and ``Time`` of the ``<PlantInformation>`` element in the ``<PlantModel>`` element that corresponds to the :doc:`DexpiModel </reference/DexpiModel/DexpiModel>` (see Proteus Schema implementation of :doc:`DexpiModel </reference/DexpiModel/DexpiModel>`\ ):

   * The date part of the :ref:`ExportDateTime <dexpi-DexpiModel-DexpiModel-ExportDateTime>` is implemented as the XML attribute ``Date`` according to the rules for the lexical representation of the XML Schema datatype `date <https://www.w3.org/TR/xmlschema-2/#date>`__ as specified by the W3C Recommendation `XML Schema Part 2: Datatypes Second Edition <https://www.w3.org/TR/xmlschema-2>`__ from October 28, 2004.

   * The time part of the :ref:`ExportDateTime <dexpi-DexpiModel-DexpiModel-ExportDateTime>` is implemented as the XML attribute ``Time`` according to the rules for the lexical representation of the XML Schema datatype `time <https://www.w3.org/TR/xmlschema-2/#time>`__ as specified by the W3C Recommendation `XML Schema Part 2: Datatypes Second Edition <https://www.w3.org/TR/xmlschema-2>`__ from October 28, 2004.

   Note that these two attribute are required by Proteus Schema, i.e., it is not possible to transfer the :ref:`null value <dexpi-concept-null-value>` :doc:`NULL_DATE_TIME </reference/DataTypes/NULL_DATE_TIME>`. Also note that the DEXPI type :doc:`DateTime </reference/DataTypes/DateTime>` does not allow timezone information. In consequence, the values of the XML attributes ``Date`` and ``Time`` must not contain timezone information.

.. admonition:: example

   December 7, 2020, 15:32:42 (:doc:`DateTime </reference/DataTypes/DateTime>` ``"2020-12-07T15:32:42"``)

   .. admonition:: proteus-example
   
      The XML fragment below only contains the ``Date`` and ``Time`` attributes of the ``<PlantInformation>`` element. This element has further required attributes (see Proteus Schema implementation of :doc:`DexpiModel </reference/DexpiModel/DexpiModel>`\ ).

      
      .. code-block:: xml
      
        <PlantModel>
          <PlantInformation
              Date="2020-12-07"
              Time="15:32:42"
              DOTS123="DOTS123">
            ...
          </PlantInformation>
        </PlantModel>
