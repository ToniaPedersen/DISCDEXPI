==========
CustomPump
==========

Overview
========

.. rubric:: Class

A :ref:`custom <dexpi-concept-custom-sibling>` :doc:`Pump </reference/Equipment/Pump>`, i.e., a :doc:`Pump </reference/Equipment/Pump>` that is not covered by any of the other subclasses of :doc:`Pump </reference/Equipment/Pump>` (:doc:`CentrifugalPump </reference/Equipment/CentrifugalPump>`, :doc:`EjectorPump </reference/Equipment/EjectorPump>`, :doc:`ReciprocatingPump </reference/Equipment/ReciprocatingPump>`, or :doc:`RotaryPump </reference/Equipment/RotaryPump>`\ ).

.. image:: CustomPump.*


.. rubric:: Supertypes

* :doc:`CustomObject </reference/Customization/CustomObject>`

* :doc:`Pump </reference/Equipment/Pump>`

.. _dexpi-Equipment-CustomPump-Displacers:

Displacers
==========

.. rubric:: Attribute (composition)


The displacers of the :doc:`CustomPump </reference/Equipment/CustomPump>`.

Multiplicity
    \*

Type
    :doc:`Displacer </reference/Equipment/Displacer>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The attribute is implemented using the :ref:`XML hierarchy of the Proteus file <proteus-concept-composition-property-xml-child>`: The  element for the attribute value (a :doc:`Displacer </reference/Equipment/Displacer>`\ ) is a child of the ``<Equipment>`` element for the attribute owner (a :doc:`CustomPump </reference/Equipment/CustomPump>`\ ).

.. admonition:: example

   .. image:: /diagrams/diagram219.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="customPump1"
            ComponentClass="CustomPump"
            ComponentClassURI="http://sandbox.dexpi.org/rdl/CustomPump" DOTS123="DOTS123">
          ...
          <Equipment
              ID="displacer1"
              ComponentClass="Displacer"
              ComponentClassURI="http://sandbox.dexpi.org/rdl/Displacer" DOTS123="DOTS123">
            ...
          <Equipment />
          ...
        <Equipment />
