.. _dexpi-Piping-PipingComponent-FluidCode:

FluidCode
=========

.. rubric:: Attribute (data)


The identification code of the fluid related to the :doc:`PipingComponent </reference/Piping/PipingComponent>`. So far, DEXPI does not define restrictions for valid values.

Multiplicity
    0..1

Type
    :doc:`NullableString </reference/DataTypes/NullableString>`

.. admonition:: proteus

   The attribute is implemented as a :ref:`DEXPI generic attribute for string values <proteus-concept-dexpi-generic-attribute-string>`.

   RDL reference
       `FLUID CODE ASSIGNMENT CLASS <http://sandbox.dexpi.org/rdl/FluidCodeAssignmentClass>`__

   Name
       FluidCodeAssignmentClass

   AttributeURI
       `http://sandbox.dexpi.org/rdl/FluidCodeAssignmentClass <http://sandbox.dexpi.org/rdl/FluidCodeAssignmentClass>`__

.. admonition:: example

   As the owning class :doc:`PipingComponent </reference/Piping/PipingComponent>` is abstract, we consider :doc:`CheckValve </reference/Piping/CheckValve>` as an arbitrary concrete subclass.

   "MNb" (:doc:`String </reference/DataTypes/String>`\ )

   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <PipingComponent
            ID="checkValve1"
            ComponentClass="CheckValve"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS292229" DOTS123="DOTS123">
          ...
          <GenericAttributes Set="DexpiAttributes" DOTS123="DOTS123">
            <GenericAttribute
              Name="FluidCodeAssignmentClass"
              AttributeURI="http://sandbox.dexpi.org/rdl/FluidCodeAssignmentClass"
              Format="string"
              Value="MNb" />
            ...
          </GenericAttributes>
          ...
        </PipingComponent>
