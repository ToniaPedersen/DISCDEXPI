          ...
        </Equipment>
      


.. _dexpi-Equipment-ColumnSection-Height:

Height
======

.. rubric:: Attribute (data)


The height of the :doc:`ColumnSection </reference/Equipment/ColumnSection>`.

Multiplicity
    0..1

Type
    :doc:`NullableLength </reference/PhysicalQuantities/NullableLength>`

.. admonition:: proteus

   The attribute is implemented as a :ref:`DEXPI generic attribute for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

   RDL reference
       `HEIGHT <http://data.posccaesar.org/rdl/RDS357704>`__

   Name
       Height

   AttributeURI
       `http://data.posccaesar.org/rdl/RDS357704 <http://data.posccaesar.org/rdl/RDS357704>`__

.. admonition:: example

   As the owning class :doc:`ColumnSection </reference/Equipment/ColumnSection>` is abstract, we consider :doc:`SubTaggedColumnSection </reference/Equipment/SubTaggedColumnSection>` as an arbitrary concrete subclass.

   The instance subTaggedColumnSection1 represents a :doc:`SubTaggedColumnSection </reference/Equipment/SubTaggedColumnSection>` with a :ref:`Height <dexpi-Equipment-ColumnSection-Height>` of 220.0 cm.

   .. image:: /diagrams/diagram158.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="subTaggedColumnSection1"
            ComponentClass="ColumnSection"
            ComponentClassURI="http://sandbox.dexpi.org/rdl/ColumnSection" DOTS123="DOTS123">
          ...
          <GenericAttributes Set="DexpiAttributes" DOTS123="DOTS123">
            <GenericAttribute
              Name="Height"
              AttributeURI="http://data.posccaesar.org/rdl/RDS357704"
              Format="double"
              Value="220.0"
              Units="Centimetre"
              UnitsURI="http://data.posccaesar.org/rdl/RDS1318004" />
            ...
          </GenericAttributes>
          ...
        </Equipment>

.. _dexpi-Equipment-ColumnSection-Internals:

Internals
=========

.. rubric:: Attribute (composition)


The :doc:`ColumnInternalsArrangement </reference/Equipment/ColumnInternalsArrangement>` of the :doc:`ColumnSection </reference/Equipment/ColumnSection>`.

Multiplicity
    0..1

Type
    :doc:`ColumnInternalsArrangement </reference/Equipment/ColumnInternalsArrangement>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The attribute is implemented using the :ref:`XML hierarchy of the Proteus file <proteus-concept-composition-property-xml-child>`: The  element for the attribute value (a :doc:`ColumnInternalsArrangement </reference/Equipment/ColumnInternalsArrangement>`\ ) is a child of the ``<Equipment>`` element for the attribute owner (a :doc:`ColumnSection </reference/Equipment/ColumnSection>`\ ).

.. admonition:: example

   As the owner type :doc:`ColumnSection </reference/Equipment/ColumnSection>` is abstract, we consider :doc:`SubTaggedColumnSection </reference/Equipment/SubTaggedColumnSection>` as an arbitrary concrete subclass. As the value type :doc:`ColumnInternalsArrangement </reference/Equipment/ColumnInternalsArrangement>` is abstract, we consider :doc:`ColumnPackingsArrangement </reference/Equipment/ColumnPackingsArrangement>` as an arbitrary concrete subclass.

   .. image:: /diagrams/diagram160.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="subTaggedColumnSection1"
            ComponentClass="ColumnSection"
            ComponentClassURI="http://sandbox.dexpi.org/rdl/ColumnSection" DOTS123="DOTS123">
          ...
          <Equipment
              ID="columnPackingsArrangement1"
              ComponentClass="ColumnPackingsArrangement"
              ComponentClassURI="http://sandbox.dexpi.org/rdl/ColumnPackingsArrangement" DOTS123="DOTS123">
            ...
          <Equipment />
          ...
        <Equipment />
