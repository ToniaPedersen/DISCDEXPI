==============
ElectricHeater
==============

Overview
========

.. rubric:: Class

A heater in which electric energy is converted into heat for useful purposes (from `http://data.posccaesar.org/rdl/RDS14070475 <http://data.posccaesar.org/rdl/RDS14070475>`__).

.. image:: ElectricHeater.*


.. rubric:: Supertypes

.. _dexpi-Equipment-ElectricHeater-DesignPower:

DesignPower
===========

.. rubric:: Attribute (data)


The power for which the :doc:`ElectricHeater </reference/Equipment/ElectricHeater>` is designed.

Multiplicity
    0..1

Type
    :doc:`NullablePower </reference/PhysicalQuantities/NullablePower>`

.. admonition:: proteus

   The attribute is implemented as a :ref:`DEXPI generic attribute for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

   RDL reference
       `DESIGN POWER <http://sandbox.dexpi.org/rdl/DesignPower>`__

   Name
       DesignPower

   AttributeURI
       `http://sandbox.dexpi.org/rdl/DesignPower <http://sandbox.dexpi.org/rdl/DesignPower>`__

.. admonition:: example

   The instance electricHeater1 represents an :doc:`ElectricHeater </reference/Equipment/ElectricHeater>` with a :ref:`DesignPower <dexpi-Equipment-ElectricHeater-DesignPower>` of 500.0 kW.

   .. image:: /diagrams/diagram256.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="electricHeater1"
            ComponentClass="ElectricHeater"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS14070475" DOTS123="DOTS123">
          ...
          <GenericAttributes Set="DexpiAttributes" DOTS123="DOTS123">
            <GenericAttribute
              Name="DesignPower"
              AttributeURI="http://sandbox.dexpi.org/rdl/DesignPower"
              Format="double"
              Value="500.0"
              Units="Kilowatt"
              UnitsURI="http://data.posccaesar.org/rdl/RDS1330919" />
            ...
          </GenericAttributes>
          ...
        </Equipment>
      


.. _dexpi-Equipment-ElectricHeater-TubeBundle:

TubeBundle

.. rubric:: Attribute (composition)


The tube bundle of the :doc:`ElectricHeater </reference/Equipment/ElectricHeater>`.

Multiplicity
    0..1

Type
    :doc:`TubeBundle </reference/Equipment/TubeBundle>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The attribute is implemented using the :ref:`XML hierarchy of the Proteus file <proteus-concept-composition-property-xml-child>`: The  element for the attribute value (a :doc:`TubeBundle </reference/Equipment/TubeBundle>`\ ) is a child of the ``<Equipment>`` element for the attribute owner (an :doc:`ElectricHeater </reference/Equipment/ElectricHeater>`\ ).
