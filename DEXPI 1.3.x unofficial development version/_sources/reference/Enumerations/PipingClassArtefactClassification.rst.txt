.. admonition:: proteus

   All data attributes with type :doc:`PipingClassArtefactClassification </reference/Enumerations/PipingClassArtefactClassification>` are implemented as :ref:`DEXPI generic attributes for enumeration values <proteus-concept-dexpi-generic-attribute-enumeration>`. In a ``<GenericAttribute>`` element, the :doc:`PipingClassArtefactClassification </reference/Enumerations/PipingClassArtefactClassification>` literal is given by means of its :ref:`RDL reference <proteus-concept-rdl-reference>` in the table above. The ``Value`` attribute of the element is the name of the RDL reference in camel-case; it equals the name of the literal. The ``ValueURI`` attribute of the element is the URI of the RDL reference.

   .. admonition:: proteus-example
   
      
      .. code-block:: xml

          Format="anyURI"

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the instance :ref:`PipingClassArtefact <dexpi-Enumerations-PipingClassArtefactClassification-PipingClassArtefact>`. For a complete example, see the attribute :ref:`PipingClassArtefact <dexpi-Piping-PipingComponent-PipingClassArtefact>` of the DEXPI class :doc:`PipingComponent </reference/Piping/PipingComponent>`.
