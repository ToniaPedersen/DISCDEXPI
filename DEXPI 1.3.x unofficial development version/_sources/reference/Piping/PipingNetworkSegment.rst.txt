====================
PipingNetworkSegment
====================

Overview
========

.. rubric:: Class

The piping limited by a Node and a Break, Node and Connector,  two Nodes, two Breaks, two Connectors or a Break and a Connector. The last five providing there are no Breaks or Connectors in between. In the last three cases the Segment will coincide with a Piping Branch (from `http://data.posccaesar.org/rdl/RDS267704 <http://data.posccaesar.org/rdl/RDS267704>`__).

.. _dexpi-Piping-PipingNetworkSegment-Connections:

Connections
===========

.. rubric:: Attribute (composition)


The connections of the :doc:`PipingNetworkSegment </reference/Piping/PipingNetworkSegment>`.

Multiplicity
    \*

Type
    :doc:`PipingConnection </reference/Piping/PipingConnection>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   In case the :doc:`PipingConnection </reference/Piping/PipingConnection>` is a :doc:`Pipe </reference/Piping/Pipe>`, the corresponding ``<CenterLine>`` element is a child of the ``<PipingNetworkSegment>``. Two ``<CenterLine>`` elements must be separated by at least one element representing a :doc:`PipingNetworkSegmentItem </reference/Piping/PipingNetworkSegmentItem>`, e.g., a ``<PipingComponent>``, in order to implement two :doc:`Pipes </reference/Piping/Pipe>`; otherwise, the two ``<CenterLine>`` elements would represent a single :doc:`Pipe </reference/Piping/Pipe>` whose graphical representation contains a visual "gap", e.g., when another :doc:`Pipe </reference/Piping/Pipe>` crosses.

   In case the :doc:`PipingConnection </reference/Piping/PipingConnection>` is a :doc:`DirectPipingConnection </reference/Piping/DirectPipingConnection>`, there is no corresponding Proteus element. A :doc:`DirectPipingConnection </reference/Piping/DirectPipingConnection>` is rather given implicitly, e.g., by two successive ``<PipingComponent>`` elements (see Proteus Schema Implementation of :doc:`DirectPipingConnection </reference/Piping/DirectPipingConnection>`\ ).

.. admonition:: example

   As the value type :doc:`PipingConnection </reference/Piping/PipingConnection>` is abstract, we consider :doc:`Pipe </reference/Piping/Pipe>` as an arbitrary concrete subclass.

   .. image:: /diagrams/diagram689.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <PipingNetworkSegment
            ID="pipingNetworkSegment1"
            ComponentClass="PipingNetworkSegment"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS267704" DOTS123="DOTS123">
          ...
          <CenterLine DOTS123="DOTS123">
            <!-- pipe1 -->
            ...
          </CenterLine>
          ...
        </PipingNetworkSegment>

.. _dexpi-Piping-PipingNetworkSegment-SourceNode:

SourceNode
==========

.. rubric:: Attribute (reference)


The :doc:`PipingNode </reference/Piping/PipingNode>` at which the :doc:`PipingNetworkSegment </reference/Piping/PipingNetworkSegment>` starts.

Multiplicity
    0..1

Type
    :doc:`PipingNode </reference/Piping/PipingNode>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The SourceNode is given by means of the ``FromNode`` XML attribute of the ``<Connection>`` element in the ``<PipingNetworkSegment>`` element. The value of the ``FromNode`` XML attribute is an integer. It refers to the zero-based index of the ``<Node>`` element within the ``<ConnectionPoints>`` element associated with the owner of the :doc:`PipingNode </reference/Piping/PipingNode>`. The owner itself is given by means of the ``FromID`` XML attribute of the ``<Connection>`` element (see Proteus Schema Implementation of :ref:`SourceItem <dexpi-Piping-PipingNetworkSegment-SourceItem>`\ ).

   For details, see :ref:`Proteus P&ID File Specification <dexpi-glossary-proteus>`. Note that in certain cases, Proteus Schema allows to omit the ``FromNode`` attribute when it is clear from the context.

.. admonition:: example

   .. image:: /diagrams/diagram696.*


   .. admonition:: proteus-example
   
      The XML fragment demonstrates the case that the SourceNode belongs to a :doc:`PipeTee </reference/Piping/PipeTee>` in another :doc:`PipingNetworkSegment </reference/Piping/PipingNetworkSegment>`.

      
      .. code-block:: xml
      
        <PipingNetworkSegment 
            ID="pipingNetworkSegment1"
            ComponentClass="PipingNetworkSegment"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS267704"
            DOTS123="DOTS123">
          ...
          <Connection FromID="pipeTee1" FromNode="2"/>
          ...
        </PipingNetworkSegment>
        ...
        <PipingNetworkSegment 
            ID="pipingNetworkSegment2"
            ComponentClass="PipingNetworkSegment"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS267704"
            DOTS123="DOTS123">
          ...
          <PipingComponent
              ID="pipeTee1"
              ComponentClass="PipeTee"
              ComponentClassURI="http://data.posccaesar.org/rdl/RDS427724"
              DOTS123="DOTS123">
            ...
            <ConnectionPoints NumPoints="4" DOTS123="DOTS123">
              <Node DOTS123="DOTS123"> ... </Node>
              <Node DOTS123="DOTS123"> ... </Node>
              <Node ID="pipingNode1" Type="process" DOTS123="DOTS123">
                <!-- This node has index 2. -->
              </Node>
              <Node DOTS123="DOTS123"> ... </Node>
            </ConnectionPoints>
            ...
          </PipingComponent>
        
          ...
        </PipingNetworkSegment>
      


.. _dexpi-Piping-PipingNetworkSegment-TargetItem:

TargetItem
==========

.. rubric:: Attribute (reference)


The item at which the PipingNetworkSegment ends.

Multiplicity
    0..1

Type
    :doc:`PipingTargetItem </reference/Piping/PipingTargetItem>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The TargetItem is given by means of the ``ToID`` XML attribute of the ``<Connection>`` element in the ``<PipingNetworkSegment>`` element. The value of the ``ToID`` XML attribute is the XML ``ID`` of the XML element corresponding to the TargetItem, e.g., a :doc:`Nozzle </reference/Equipment/Nozzle>` or a :doc:`PipingComponent </reference/Piping/PipingComponent>`.

.. admonition:: example

   .. image:: /diagrams/diagram697.*


   .. admonition:: proteus-example
   
      The XML fragment demonstrates the case that the TargetItem of the ``<PipingNetworkSegment>`` is a :doc:`Nozzle </reference/Equipment/Nozzle>` of some :doc:`Equipment </reference/Equipment/Equipment>` item.

      
      .. code-block:: xml
      
        <PipingNetworkSegment 
            ID="pipingNetworkSegment1"
            ComponentClass="PipingNetworkSegment"
            ComponentClassURI="http://data.posccaesar.org/rdl/RDS267704"
            DOTS123="DOTS123">
          ...
          <Connection ToID="nozzle1" DOTS123="DOTS123">
          ...
        </PipingNetworkSegment>
        ...
        <Equipment DOTS123="DOTS123">
          ...

              ID="nozzle1"
              ComponentClass="Nozzle"
              ComponentClassURI="http://data.posccaesar.org/rdl/RDS415214"
              DOTS123="DOTS123">
            ...
          </Nozzle>
          ...
        </Equipment>
