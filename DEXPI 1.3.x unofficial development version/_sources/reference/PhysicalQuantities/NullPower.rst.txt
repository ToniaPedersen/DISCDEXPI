=========
NullPower
=========

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullablePower </reference/PhysicalQuantities/NullablePower>`. The only instance of this singleton type is :doc:`NULL_POWER </reference/PhysicalQuantities/NULL_POWER>`.

.. image:: NullPower.*


.. rubric:: Supertypes

* :doc:`NullablePower </reference/PhysicalQuantities/NullablePower>`

.. admonition:: proteus

   All data attributes with type :doc:`NullablePower </reference/PhysicalQuantities/NullablePower>` (the base type of :doc:`NullPower </reference/PhysicalQuantities/NullPower>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram597.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_POWER </reference/PhysicalQuantities/NULL_POWER>` (e.g., see the implementation example for the attribute :ref:`DesignShaftPower <dexpi-Equipment-Agglomerator-DesignShaftPower>` of the DEXPI class :doc:`Agglomerator </reference/Equipment/Agglomerator>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
