============
CustomFilter
============

Overview
========

.. rubric:: Class

A :ref:`custom <dexpi-concept-custom-sibling>` :doc:`Filter </reference/Equipment/Filter>`, i.e., a :doc:`Filter </reference/Equipment/Filter>` that is not covered by any of the other subclasses of :doc:`Filter </reference/Equipment/Filter>` (:doc:`GasFilter </reference/Equipment/GasFilter>` or :doc:`LiquidFilter </reference/Equipment/LiquidFilter>`\ ).

.. image:: CustomFilter.*

.. _dexpi-Equipment-CustomFilter-FilterUnit:

FilterUnit
==========

.. rubric:: Attribute (composition)


The filter unit of the :doc:`CustomFilter </reference/Equipment/CustomFilter>`.

Multiplicity
    0..1

Type
    :doc:`FilterUnit </reference/Equipment/FilterUnit>`

Opposite multiplicity
    0..1

.. admonition:: proteus

   The attribute is implemented using the :ref:`XML hierarchy of the Proteus file <proteus-concept-composition-property-xml-child>`: The  element for the attribute value (a :doc:`FilterUnit </reference/Equipment/FilterUnit>`\ ) is a child of the ``<Equipment>`` element for the attribute owner (a :doc:`CustomFilter </reference/Equipment/CustomFilter>`\ ).

.. admonition:: example

   .. image:: /diagrams/diagram207.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <Equipment
            ID="customFilter1"
            ComponentClass="CustomFilter"
            ComponentClassURI="http://sandbox.dexpi.org/rdl/CustomFilter" DOTS123="DOTS123">
          ...
          <Equipment
              ID="filterUnit1"
              ComponentClass="FilterUnit"
              ComponentClassURI="http://sandbox.dexpi.org/rdl/FilterUnit" DOTS123="DOTS123">
            ...
          <Equipment />
          ...
        <Equipment />
