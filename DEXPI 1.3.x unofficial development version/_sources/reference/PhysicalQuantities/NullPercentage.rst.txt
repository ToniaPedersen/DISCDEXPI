==============
NullPercentage
==============

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullablePercentage </reference/PhysicalQuantities/NullablePercentage>`. The only instance of this singleton type is :doc:`NULL_PERCENTAGE </reference/PhysicalQuantities/NULL_PERCENTAGE>`.

.. image:: NullPercentage.*


.. rubric:: Supertypes

* :doc:`NullablePercentage </reference/PhysicalQuantities/NullablePercentage>`

.. admonition:: proteus

   All data attributes with type :doc:`NullablePercentage </reference/PhysicalQuantities/NullablePercentage>` (the base type of :doc:`NullPercentage </reference/PhysicalQuantities/NullPercentage>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram596.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_PERCENTAGE </reference/PhysicalQuantities/NULL_PERCENTAGE>` (e.g., see the implementation example for the attribute :ref:`Efficiency <dexpi-Equipment-FilterUnit-Efficiency>` of the DEXPI class :doc:`FilterUnit </reference/Equipment/FilterUnit>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
