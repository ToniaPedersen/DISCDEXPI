=======================
ElectricalFrequencyUnit
=======================

Overview
========

.. rubric:: Enumeration

A unit of measurement for a physical quantity of application type :doc:`NullableElectricalFrequency </reference/PhysicalQuantities/NullableElectricalFrequency>` with dimension T\ :sup:`-1`.

.. admonition:: proteus

   :doc:`ElectricalFrequencyUnit </reference/PhysicalQuantities/ElectricalFrequencyUnit>` is only used as the type of the :ref:`Unit <dexpi-PhysicalQuantities-ElectricalFrequency-Unit>` attribute of :doc:`ElectricalFrequency </reference/PhysicalQuantities/ElectricalFrequency>`. :doc:`ElectricalFrequency </reference/PhysicalQuantities/ElectricalFrequency>` is implemented using :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`. In a ``<GenericAttribute>`` element, the :doc:`ElectricalFrequencyUnit </reference/PhysicalQuantities/ElectricalFrequencyUnit>` literal is given by means of its :ref:`RDL reference <proteus-concept-rdl-reference>` in the table above. The ``Units`` attribute of the element is the name of the RDL reference in camel-case; it equals the name of the literal. The ``UnitsURI`` attribute of the element is the URI of the RDL reference.
