=====================
NumberPerTimeInterval
=====================

Overview
========

.. rubric:: Data type

An :ref:`actual value <dexpi-concept-actual-value>` for a physical quantity of type :doc:`NullableNumberPerTimeInterval </reference/PhysicalQuantities/NullableNumberPerTimeInterval>`, i.e., a physical quantity that has a numerical value and a unit of measurement.

.. admonition:: proteus

   All data attributes with type :doc:`NullableNumberPerTimeInterval </reference/PhysicalQuantities/NullableNumberPerTimeInterval>` (the base type of :doc:`NumberPerTimeInterval </reference/PhysicalQuantities/NumberPerTimeInterval>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   The instance numberPerTimeInterval1 represents a :doc:`NumberPerTimeInterval </reference/PhysicalQuantities/NumberPerTimeInterval>` of 42.0 s\ :sup:`-1`.

   .. image:: /diagrams/diagram605.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double"
          Value="42.0"
          Units="ReciprocalSecond"
          UnitsURI="http://data.posccaesar.org/rdl/RDS1355489" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the instance numberPerTimeInterval1. For a complete example, see the attribute :ref:`DesignCapacityWeighingQuantities <dexpi-Equipment-BatchWeigher-DesignCapacityWeighingQuantities>` of the DEXPI class :doc:`BatchWeigher </reference/Equipment/BatchWeigher>`.
