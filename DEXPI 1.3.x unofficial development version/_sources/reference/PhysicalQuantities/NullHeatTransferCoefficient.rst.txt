===========================
NullHeatTransferCoefficient
===========================

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullableHeatTransferCoefficient </reference/PhysicalQuantities/NullableHeatTransferCoefficient>`. The only instance of this singleton type is :doc:`NULL_HEAT_TRANSFER_COEFFICIENT </reference/PhysicalQuantities/NULL_HEAT_TRANSFER_COEFFICIENT>`.

.. image:: NullHeatTransferCoefficient.*


.. rubric:: Supertypes

* :doc:`NullableHeatTransferCoefficient </reference/PhysicalQuantities/NullableHeatTransferCoefficient>`

.. admonition:: proteus

   All data attributes with type :doc:`NullableHeatTransferCoefficient </reference/PhysicalQuantities/NullableHeatTransferCoefficient>` (the base type of :doc:`NullHeatTransferCoefficient </reference/PhysicalQuantities/NullHeatTransferCoefficient>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram591.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_HEAT_TRANSFER_COEFFICIENT </reference/PhysicalQuantities/NULL_HEAT_TRANSFER_COEFFICIENT>` (e.g., see the implementation example for the attribute :ref:`DesignHeatTransferCoefficient <dexpi-Equipment-HeatExchanger-DesignHeatTransferCoefficient>` of the DEXPI class :doc:`HeatExchanger </reference/Equipment/HeatExchanger>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
