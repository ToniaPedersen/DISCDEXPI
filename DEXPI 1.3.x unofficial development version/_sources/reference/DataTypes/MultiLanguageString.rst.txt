===================
MultiLanguageString
===================

Overview
========

.. rubric:: Data type

A container for one or more :doc:`SingleLanguageString </reference/DataTypes/SingleLanguageString>`\ sDOTANDSPACEHACK\ :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>` is used as the type of data attributes which have language-dependent string values: Each :doc:`SingleLanguageString </reference/DataTypes/SingleLanguageString>` contains a :doc:`NullableString </reference/DataTypes/NullableString>` :ref:`Value <dexpi-DataTypes-SingleLanguageString-Value>` and a :ref:`Language <dexpi-DataTypes-SingleLanguageString-Language>` tag.

.. admonition:: proteus

   The Proteus Schema implementation of :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>` depends on the Proteus Schema implementations of the data attributes with type :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>`.

   * :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>` is the type of the :ref:`Value <dexpi-Customization-CustomMultiLanguageStringAttribute-Value>` attribute of :doc:`CustomMultiLanguageStringAttribute </reference/Customization/CustomMultiLanguageStringAttribute>`. :doc:`CustomMultiLanguageStringAttribute </reference/Customization/CustomMultiLanguageStringAttribute>` is implemented as a :ref:`set of custom generic attributes for multi-language string values <proteus-concept-dexpi-custom-generic-attribute-multi-language-string>`.

   * All other data attributes with type :doc:`MultiLanguageString </reference/DataTypes/MultiLanguageString>` are implemented as :ref:`sets of DEXPI generic attributes for multi-language string values <proteus-concept-dexpi-generic-attribute-multi-language-string>`. For example, see the :ref:`ChamberDescription <dexpi-Equipment-Chamber-ChamberDescription>` attribute of :doc:`Chamber </reference/Equipment/Chamber>`.
