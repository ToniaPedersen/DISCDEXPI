=========
NullForce
=========

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullableForce </reference/PhysicalQuantities/NullableForce>`. The only instance of this singleton type is :doc:`NULL_FORCE </reference/PhysicalQuantities/NULL_FORCE>`.

.. image:: NullForce.*


.. rubric:: Supertypes

* :doc:`NullableForce </reference/PhysicalQuantities/NullableForce>`

.. admonition:: proteus

   All data attributes with type :doc:`NullableForce </reference/PhysicalQuantities/NullableForce>` (the base type of :doc:`NullForce </reference/PhysicalQuantities/NullForce>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram590.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_FORCE </reference/PhysicalQuantities/NULL_FORCE>` (e.g., see the implementation example for the attribute :ref:`LowerLimitDesignPressingForce <dexpi-Equipment-ReciprocatingPressureAgglomerator-LowerLimitDesignPressingForce>` of the DEXPI class :doc:`ReciprocatingPressureAgglomerator </reference/Equipment/ReciprocatingPressureAgglomerator>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
