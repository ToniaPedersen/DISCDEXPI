.. admonition:: proteus

   All data attributes with type :doc:`PipingClassBreakClassification </reference/Enumerations/PipingClassBreakClassification>` are implemented as :ref:`DEXPI generic attributes for enumeration values <proteus-concept-dexpi-generic-attribute-enumeration>`. In a ``<GenericAttribute>`` element, the :doc:`PipingClassBreakClassification </reference/Enumerations/PipingClassBreakClassification>` literal is given by means of its :ref:`RDL reference <proteus-concept-rdl-reference>` in the table above. The ``Value`` attribute of the element is the name of the RDL reference in camel-case; it equals the name of the literal. The ``ValueURI`` attribute of the element is the URI of the RDL reference.

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the instance :ref:`PipingClassBreak <dexpi-Enumerations-PipingClassBreakClassification-PipingClassBreak>`. For a complete example, see the attribute :ref:`PipingClassBreak <dexpi-Piping-PropertyBreak-PipingClassBreak>` of the DEXPI class :doc:`PropertyBreak </reference/Piping/PropertyBreak>`.
