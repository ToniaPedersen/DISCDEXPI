==========
NullLength
==========

Overview
========

.. rubric:: Data type

A :ref:`null value <dexpi-concept-null-value>` for a physical quantity of type :doc:`NullableLength </reference/PhysicalQuantities/NullableLength>`. The only instance of this singleton type is :doc:`NULL_LENGTH </reference/PhysicalQuantities/NULL_LENGTH>`.

.. image:: NullLength.*


.. rubric:: Supertypes

* :doc:`NullableLength </reference/PhysicalQuantities/NullableLength>`

.. admonition:: proteus

   All data attributes with type :doc:`NullableLength </reference/PhysicalQuantities/NullableLength>` (the base type of :doc:`NullLength </reference/PhysicalQuantities/NullLength>`\ ) are implemented as :ref:`DEXPI generic attributes for physical quantities <proteus-concept-dexpi-generic-attribute-physical-quantity>`.

.. admonition:: example

   .. image:: /diagrams/diagram592.*


   .. admonition:: proteus-example
   
      
      .. code-block:: xml
      
        <GenericAttribute
          Format="double" DOTS123="DOTS123">
      

      Note that the ``<GenericAttribute>`` element must have a ``Name`` and an ``AttributeURI`` attribute. They depend on the data type attribute of the DEXPI class that owns the null instance :doc:`NULL_LENGTH </reference/PhysicalQuantities/NULL_LENGTH>` (e.g., see the implementation example for the attribute :ref:`Diameter <dexpi-Equipment-AgitatorRotor-Diameter>` of the DEXPI class :doc:`AgitatorRotor </reference/Equipment/AgitatorRotor>`\ )DOTANDSPACEHACKFor a null value, the attributes ``Units``, ``UnitsURI``, and ``Value`` *must not be used*.
